# Implementation Plan: Kalman Filter State Estimation Engine
Date: 2026-03-28
Status: Draft

## Summary

Implement the seven stub functions in `backend/kalman.py` to provide a fully operational
Unscented Kalman Filter (UKF) state estimator, one instance per tracked object. Each filter
maintains a 6-element ECI J2000 state vector and 6x6 covariance matrix, performs
predict+update cycles driven by new TLE observations, computes NIS for anomaly detection,
and supports recalibration when the filter diverges. This plan also replaces the
`pytest.skip` stubs in `tests/test_kalman.py` with working assertions and adds additional
edge-case tests to reach the NF-031 coverage target (>=70%).

## Requirements addressed

- **F-020 [DEMO]**: UKF state estimate per tracked object, updated with each new TLE observation.
- **F-021**: Each update cycle = predict step (propagate forward) + update step (incorporate observation).
- **F-022**: NIS statistic computed for each filter update.
- **F-023**: Covariance matrix maintained per object across updates.
- **F-024**: Process noise matrix Q configurable per object class (debris, active satellite, rocket body).
- **F-025**: Measurement noise matrix R configurable, reflecting TLE positional uncertainty.
- **NF-001** (partial): UKF update for a single object must complete within 100ms.
- **NF-030** (partial): All public functions have type annotations and docstrings (stubs already satisfy this; implementations must preserve them).
- **NF-031** (partial): Unit test coverage >=70% on `kalman.py`.

## Files affected

- `backend/kalman.py` — Implement all seven stubs: `init_filter`, `predict`, `update`,
  `compute_nis`, `get_state`, `recalibrate`, `compute_confidence`. Add two module-level
  constants for default Q matrices (per object class) and a module-level default R matrix.
  Add a private helper `_make_default_covariance_p0` for initial covariance construction.
  Add a module-level constant `OBJECT_CLASS_Q` (dict mapping class name to 6x6 array) and
  `DEFAULT_R` (6x6 array).
- `tests/test_kalman.py` — Replace all eight `pytest.skip` stubs with working assertions.
  Add seven additional tests for edge cases and full-cycle integration.

## Data flow changes

**Before:** All `kalman.py` functions raise `NotImplementedError`. No filter state can be
created or updated.

**After:**
```
New TLE arrives (from ingest.py event)
        |
        v
propagator.tle_epoch_utc(line1) --> target_epoch_utc
        |
        v
kalman.predict(filter_state, target_epoch_utc, line1, line2)
  |
  +--> builds closure fx over (line1, line2)
  +--> UKF.predict(fx=fx)   [FilterPy: propagates 13 sigma points via SGP4]
  +--> returns predicted_state_eci_km (6-element array)
        |
        v
propagator.tle_to_state_vector_eci_km(line1, line2, target_epoch_utc)
        --> observation_eci_km (6-element array, ECI J2000)
        |
        v
kalman.update(filter_state, observation_eci_km, target_epoch_utc)
  |
  +--> UKF.update(z=observation_eci_km, hx=identity_hx)
  +--> kalman.compute_nis(innovation, S)
  +--> sets filter_state['nis'], filter_state['innovation_eci_km']
  +--> sets filter_state['anomaly_flag'] = (NIS > CHI2_THRESHOLD_6DOF)
  +--> appends NIS to filter_state['nis_history']
  +--> kalman.compute_confidence(nis, nis_history) --> filter_state['confidence']
  +--> returns updated filter_state dict
        |
        v
Caller (main.py / anomaly.py) reads filter_state:
  - filter_state['nis'] --> fed to anomaly.py for event classification
  - filter_state['anomaly_flag'] --> if True, caller invokes kalman.recalibrate(...)
  - filter_state['state_eci_km'] and filter_state['covariance_km2'] --> stored to SQLite
  - full filter_state dict --> serialised to WebSocket message via get_state()
```

**Boundary clarification (architecture 3.3 vs. anomaly.py):**
`kalman.py` computes NIS and sets `anomaly_flag` when NIS exceeds `CHI2_THRESHOLD_6DOF`.
`kalman.py` provides `recalibrate()` but does NOT call it autonomously. The caller
(expected to be `main.py` or a future `anomaly.py` integration layer) is responsible for
reading `anomaly_flag` and deciding whether to invoke `recalibrate`. This matches the
architecture diagram (section 4) where `anomaly.py` is a distinct downstream component
fed by `kalman.py` output. F-030 and F-033 are therefore partially satisfied here
(NIS threshold comparison and recalibration capability) and fully satisfied when
`anomaly.py` is implemented.

## Implementation steps

### Phase 1: Module-level constants and default matrices

1. **Add object-class Q constants and default R matrix** (`backend/kalman.py`)
   - Action: Below the existing `CHI2_THRESHOLD_6DOF` constant, add the following
     module-level definitions. Do not alter any existing lines.

     ```python
     # Object class identifiers for Q matrix selection (F-024)
     OBJECT_CLASS_DEBRIS: str = "debris"
     OBJECT_CLASS_ACTIVE: str = "active_satellite"
     OBJECT_CLASS_ROCKET_BODY: str = "rocket_body"

     # Process noise matrices Q per object class (F-024).
     # Units: km^2 for position blocks, (km/s)^2 for velocity blocks.
     # Hand-tuned for POC; see POST-002 for adaptive noise estimation.
     # Position variance: diagonal scaled by assumed unmodeled acceleration over
     # a 30-minute update interval. Velocity variance: smaller, reflecting that
     # unmodeled forces accumulate primarily in velocity over short intervals.
     #
     # Debris: higher drag uncertainty, no maneuver capability.
     #   sigma_pos ~ 1 km, sigma_vel ~ 0.01 km/s
     # Active satellite: moderate uncertainty + maneuver probability.
     #   sigma_pos ~ 0.5 km, sigma_vel ~ 0.05 km/s
     # Rocket body: intermediate drag uncertainty, inert.
     #   sigma_pos ~ 0.75 km, sigma_vel ~ 0.02 km/s
     _Q_DEBRIS: NDArray[np.float64] = np.diag(
         [1.0, 1.0, 1.0, 1e-4, 1e-4, 1e-4]
     ).astype(np.float64)

     _Q_ACTIVE_SATELLITE: NDArray[np.float64] = np.diag(
         [0.25, 0.25, 0.25, 25e-4, 25e-4, 25e-4]
     ).astype(np.float64)

     _Q_ROCKET_BODY: NDArray[np.float64] = np.diag(
         [0.5625, 0.5625, 0.5625, 4e-4, 4e-4, 4e-4]
     ).astype(np.float64)

     OBJECT_CLASS_Q: dict[str, NDArray[np.float64]] = {
         OBJECT_CLASS_DEBRIS: _Q_DEBRIS,
         OBJECT_CLASS_ACTIVE: _Q_ACTIVE_SATELLITE,
         OBJECT_CLASS_ROCKET_BODY: _Q_ROCKET_BODY,
     }

     # Default measurement noise matrix R (F-025).
     # TLE positional uncertainty: ~1 km (1-sigma) for publicly available TLEs.
     # Velocity uncertainty: ~0.01 km/s derived from TLE mean-motion precision.
     # These are diagonal; off-diagonal correlation is neglected for POC.
     DEFAULT_R: NDArray[np.float64] = np.diag(
         [1.0, 1.0, 1.0, 1e-4, 1e-4, 1e-4]
     ).astype(np.float64)
     ```

   - Why: These constants are referenced by `init_filter` and `recalibrate`. Defining them
     at module level keeps them visible for tests and for callers that need to override per
     object. The Q values are physically motivated (not arbitrary): 1 km position noise over
     a 30-minute interval corresponds to unmodeled accelerations on the order of 1e-6 km/s^2,
     which is reasonable for LEO debris.
   - Dependencies: none.
   - Risk: **Low**. The numerical values are approximate and explicitly labelled as
     hand-tuned (per architecture doc). The implementer must not re-derive these values;
     use exactly the numbers specified above.

2. **Add `_make_default_covariance_p0` private helper** (`backend/kalman.py`)
   - Action: Add the following private function after the module-level constants:

     ```python
     def _make_default_covariance_p0(
         inflation_factor: float = 1.0,
     ) -> NDArray[np.float64]:
         """Construct a default initial covariance matrix P0.

         Uses diagonal entries corresponding to ~10 km position uncertainty
         and ~0.1 km/s velocity uncertainty (1-sigma), inflatable by a scalar
         factor for recalibration.

         Args:
             inflation_factor: Multiply all diagonal entries by this factor.

         Returns:
             6x6 diagonal covariance matrix in km^2 and (km/s)^2.
         """
         p0_diag = np.array([100.0, 100.0, 100.0, 0.01, 0.01, 0.01],
                            dtype=np.float64)
         return np.diag(p0_diag * inflation_factor)
     ```

   - Why: Initial covariance is needed by both `init_filter` (first initialisation) and
     `recalibrate` (post-divergence reset). Centralising it avoids duplication and ensures
     recalibration uses the same base P0 as initial construction. 10 km / 0.1 km/s is a
     deliberately large initial uncertainty, consistent with treating the first TLE as a
     coarse prior before the filter has converged.
   - Dependencies: Requires step 1 (module-level constants must already be defined above it).
   - Risk: **Low**

### Phase 2: `init_filter` implementation

3. **Implement `init_filter`** (`backend/kalman.py`)
   - Action: Replace the `raise NotImplementedError("not implemented")` body with the
     following logic (preserve the existing docstring and signature exactly):

     a. Validate `epoch_utc` is UTC-aware. If `epoch_utc.tzinfo is None`, raise
        `ValueError("epoch_utc must be UTC-aware")`.

     b. Validate `state_eci_km` has shape `(6,)`. If not, raise
        `ValueError(f"state_eci_km must have shape (6,), got {state_eci_km.shape}")`.

     c. Resolve `q_matrix` and `r_matrix`:
        - If `process_noise_q` is `None`, use `OBJECT_CLASS_Q[OBJECT_CLASS_ACTIVE]`
          as the default (conservative choice).
        - Otherwise use `process_noise_q`.
        - If `measurement_noise_r` is None, use `DEFAULT_R`.
        - Otherwise use `measurement_noise_r`.

     d. Instantiate FilterPy UKF:
        ```python
        from filterpy.kalman import UnscentedKalmanFilter
        from filterpy.kalman import MerweScaledSigmaPoints

        points = MerweScaledSigmaPoints(n=6, alpha=1e-3, beta=2.0, kappa=0.0)

        ukf = UnscentedKalmanFilter(
            dim_x=6,
            dim_z=6,
            dt=1.0,          # nominal dt in seconds; overridden in each predict call
            hx=_identity_hx, # measurement function (see step below)
            fx=None,         # placeholder; set per predict call via closure
            points=points,
        )
        ukf.x = state_eci_km.copy().astype(np.float64)
        ukf.P = _make_default_covariance_p0()
        ukf.Q = q_matrix.copy()
        ukf.R = r_matrix.copy()
        ```

     e. Return the filter state dict:
        ```python
        return {
            "filter": ukf,
            "last_epoch_utc": epoch_utc,
            "q_matrix": q_matrix.copy(),
            "r_matrix": r_matrix.copy(),
            "nis": 0.0,
            "nis_history": [],        # list[float], appended on each update
            "innovation_eci_km": np.zeros(6, dtype=np.float64),
            "anomaly_flag": False,
            "confidence": 1.0,
            "state_eci_km": state_eci_km.copy().astype(np.float64),
            "covariance_km2": ukf.P.copy(),
        }
        ```

   - **Additional requirement for this step**: Add the private `_identity_hx` function
     (before `init_filter` in the file) that FilterPy will call during UKF update:

     ```python
     def _identity_hx(state_eci_km: NDArray[np.float64]) -> NDArray[np.float64]:
         """Measurement function: observation is the full state vector (identity).

         For POC, TLE-derived state vectors are direct observations of the full
         ECI state. No partial observability.

         Args:
             state_eci_km: 6-element state vector.

         Returns:
             The state vector unchanged.
         """
         return state_eci_km
     ```

   - Why: FilterPy's `UnscentedKalmanFilter` requires `hx` and `fx` at construction.
     `hx` is the identity because we observe the full state (position+velocity) from the
     TLE. `fx` is set to `None` at construction and overridden per `predict` call via
     a closure (explained in Phase 3). The `MerweScaledSigmaPoints` with `alpha=1e-3`,
     `beta=2.0`, `kappa=0.0` are the standard UKF tuning parameters for a 6-state
     Gaussian distribution (per Wan & van der Merwe, 2000).
   - Dependencies: Requires steps 1 and 2.
   - Risk: **Medium**. FilterPy's `UnscentedKalmanFilter` initialisation API must be
     verified against filterpy>=1.4.5. The `fx=None` at construction is non-standard;
     see Phase 3 for how `fx` is set before each predict call. If FilterPy raises on
     `fx=None`, pass a lambda identity instead and override in `predict`.

### Phase 3: `predict` implementation

4. **Implement `predict`** (`backend/kalman.py`)
   - Action: Add `from backend import propagator` (or `import backend.propagator as propagator`
     — see dependency note below) at the top of the imports block. Then replace the
     `raise NotImplementedError` body of `predict` with:

     a. Validate `target_epoch_utc` is UTC-aware. Raise `ValueError` if naive.

     b. Validate `filter_state` has the required keys: `"filter"`, `"last_epoch_utc"`.
        Raise `KeyError` with a descriptive message if missing.

     c. Compute `dt_s` (elapsed seconds from last epoch to target epoch):
        ```python
        dt_s: float = (
            target_epoch_utc - filter_state["last_epoch_utc"]
        ).total_seconds()
        ```
        If `dt_s <= 0`, raise `ValueError("target_epoch_utc must be after last_epoch_utc")`.

     d. Build the process function `fx` as a closure over `tle_line1`, `tle_line2`, and
        `target_epoch_utc`. FilterPy calls `fx(x, dt)` for each sigma point where `x` is
        a 6-element state vector and `dt` is a scalar (seconds). The closure ignores `dt`
        because the target epoch is fixed, and uses the TLE-based propagator directly:

        ```python
        def _fx_sgp4(x: NDArray[np.float64], dt: float) -> NDArray[np.float64]:
            """Process model: propagate sigma point via SGP4 to target_epoch_utc."""
            # x contains [x_km, y_km, z_km, vx_km_s, vy_km_s, vz_km_s] from
            # the UKF sigma-point distribution. For SGP4 as the process model,
            # we propagate using the TLE (not the sigma-point state), because SGP4
            # is a trajectory model, not a force model — see design note below.
            return propagator.tle_to_state_vector_eci_km(
                tle_line1, tle_line2, target_epoch_utc
            )
        ```

        **Design note on sigma-point propagation via SGP4**: In a standard UKF, `fx`
        perturbs each sigma point through the dynamic model and produces 13 different
        predicted states. However, SGP4 is a deterministic trajectory model parameterised
        by TLE elements, not a force-model ODE. All 13 sigma points would map to the
        same SGP4 state vector because SGP4 does not accept an arbitrary initial state
        as input — it only propagates from its own TLE-encoded initial conditions.
        Therefore, `_fx_sgp4` returns the same SGP4-propagated state for all sigma points.
        The covariance growth from `P_predict = F*P*F^T + Q` is dominated by Q in this
        case, which is the physically correct behaviour: uncertainty grows between
        observations due to unmodeled forces (captured by Q), not due to sigma-point
        divergence through the dynamics.
        This is a known limitation of using SGP4 as a UKF process model (as opposed to
        a numerical integrator). Document this in a comment at the function.

     e. Assign `filter_state["filter"].fx = _fx_sgp4` on the UKF object before calling
        predict.

     f. Call `filter_state["filter"].predict(dt=dt_s)`.

     g. Update the filter state dict but do NOT yet update `last_epoch_utc` (that happens
        in `update` after the observation is incorporated):
        ```python
        filter_state["state_eci_km"] = filter_state["filter"].x.copy()
        filter_state["covariance_km2"] = filter_state["filter"].P.copy()
        ```

     h. Return `filter_state["filter"].x.copy()` (the predicted state, 6-element array).

   - **Import note**: `kalman.py` must import `propagator` as a relative import within
     the `backend` package. Use `from . import propagator` if running as part of the
     package, or `from backend import propagator` if running from the repo root. The
     implementer should verify the existing import style in `main.py` or other backend
     files and match it. If there is no existing pattern, use `from . import propagator`.
   - Why: The UKF predict step advances the state distribution forward in time. Using SGP4
     as the process model is the correct approach given the TLE-based observation model.
     The closure pattern is required because FilterPy's sigma-point propagation calls `fx`
     per point, and the TLE context must be bound at predict-time without modifying the
     `predict` function signature (which is already fixed in the stub).
   - Dependencies: Requires step 3 (init_filter, so the dict schema is established).
     Requires `propagator.py` to be implemented (plan: 2026-03-28-propagator.md). If
     `propagator.py` is still a stub, tests must mock `propagator.tle_to_state_vector_eci_km`.
   - Risk: **High**. The sigma-point-all-return-same-state design is a deliberate
     simplification. It means the UKF predict step does not spread sigma points through
     a force model and the predicted covariance grows only via `Q`. This is acceptable for
     POC but is a known fidelity limitation. Flag for post-POC with a `# POST-002`
     comment at the `_fx_sgp4` definition.

### Phase 4: `compute_nis` implementation

5. **Implement `compute_nis`** (`backend/kalman.py`)
   - Action: Replace `raise NotImplementedError` with:

     ```python
     s_inv = np.linalg.inv(innovation_covariance_km2)
     nis_val: float = float(innovation_eci_km @ s_inv @ innovation_eci_km)
     return nis_val
     ```

   - **Numerical safety**: Add a check before inversion. If
     `np.linalg.matrix_rank(innovation_covariance_km2) < 6`, the matrix is singular.
     In that case, log a warning and use `np.linalg.pinv` instead:

     ```python
     if np.linalg.matrix_rank(innovation_covariance_km2) < 6:
         # DEVIATION note location: if this branch is ever reached in testing,
         # flag it. S should be positive definite if Q and R are positive definite.
         s_inv = np.linalg.pinv(innovation_covariance_km2)
     else:
         s_inv = np.linalg.inv(innovation_covariance_km2)
     ```

   - Why: The NIS formula `y^T * S^{-1} * y` requires matrix inversion of S (innovation
     covariance). S = H*P*H^T + R; since H is identity and R is positive definite, S
     should always be positive definite. However, numerical errors can cause near-singular
     S in edge cases. Using `pinv` as fallback prevents crashes at the cost of a slightly
     incorrect NIS value in degenerate cases.
   - Dependencies: none (pure numpy, no filter state needed).
   - Risk: **Low**

### Phase 5: `update` implementation

6. **Implement `update`** (`backend/kalman.py`)
   - Action: Replace `raise NotImplementedError` with:

     a. Validate `epoch_utc` is UTC-aware. Raise `ValueError` if naive.

     b. Validate `observation_eci_km` has shape `(6,)`. Raise `ValueError` if not.

     c. Run the FilterPy UKF update step:
        ```python
        ukf = filter_state["filter"]
        ukf.update(observation_eci_km.astype(np.float64))
        ```

     d. Extract the innovation vector and innovation covariance from the UKF internals.
        FilterPy stores the innovation as `ukf.y` and the innovation covariance as `ukf.S`

        ```python

        innovation_covariance_km2: NDArray[np.float64] = ukf.S.copy()
        ```

     e. Compute NIS:
        ```python

        ```

     f. Update the filter state dict:
        ```python
        filter_state["innovation_eci_km"] = innovation_eci_km
        filter_state["nis"] = nis_val
        filter_state["last_epoch_utc"] = epoch_utc
        filter_state["state_eci_km"] = ukf.x.copy()

        ```

     g. Append NIS to history (cap history at last 20 values to bound memory):
        ```python
        filter_state["nis_history"].append(nis_val)
        if len(filter_state["nis_history"]) > 20:
            filter_state["nis_history"] = filter_state["nis_history"][-20:]
        ```

     h. Compute confidence:
        ```python
        filter_state["confidence"] = compute_confidence(
            nis_val, filter_state["nis_history"]
        )
        ```

     i. Return `filter_state` (mutated in place AND returned, consistent with the stub
        return type annotation).

   - **FilterPy attribute verification**: The implementer must verify that FilterPy
     `UnscentedKalmanFilter.update()` stores `y` (innovation) and `S` (innovation
     covariance) as instance attributes after the call. As of filterpy 1.4.5, the UKF
     stores `self.y` and `self.S` during `update()`. If the installed version differs,
     compute innovation manually as `observation_eci_km - ukf.x_prior` and S from
     `ukf.P_prior + ukf.R`.
   - Dependencies: Requires steps 3 (init_filter) and 5 (compute_nis).
   - Risk: **Medium**. FilterPy's internal attribute naming must be confirmed against the
     installed version. This is the most FilterPy-version-sensitive step in the plan.

     b. Preserve `q_matrix` and `r_matrix` from the diverged filter state so the

        ```python
        q_matrix = filter_state["q_matrix"]
        r_matrix = filter_state["r_matrix"]
        ```

     c. Call `init_filter` with the new observation as the initial state and inflated P0:

        ```python

        ```

     e. Return `new_state`.

     `anomaly.py` to see the full history including the divergence spike for classification.
   - Dependencies: Requires steps 1, 2, 3 (init_filter must be implemented first).
   - Risk: **Low**

### Phase 8: `compute_confidence` implementation

9. **Implement `compute_confidence`** (`backend/kalman.py`)
   - Action: Replace `raise NotImplementedError` with:

     ```python
     # Confidence is based on the fraction of recent NIS values below threshold.
     # Current NIS is weighted 2x relative to history entries.
     # Returns 1.0 if all recent values are well below threshold,
     # 0.0 if current NIS far exceeds threshold.
     if not nis_history:
         # No history yet: use only current NIS
         history_score: float = 1.0
     else:
         below = sum(1.0 for v in nis_history if v <= CHI2_THRESHOLD_6DOF)
         history_score = below / len(nis_history)

     # Current NIS score: map NIS to [0, 1] with threshold as the midpoint.
     # NIS=0 -> 1.0, NIS=threshold -> 0.5, NIS=2*threshold -> ~0.0
     current_score: float = max(0.0, 1.0 - (nis / (2.0 * CHI2_THRESHOLD_6DOF)))

     # Weighted blend: current NIS has 2x weight vs. history average
     confidence: float = (2.0 * current_score + history_score) / 3.0
     return float(np.clip(confidence, 0.0, 1.0))
     ```

   - Why: The confidence score must map to the frontend colour thresholds from F-051:
     green (>0.85), amber (0.60–0.85), red (<0.60). The formula above gives:
     - Consistent filter (NIS ~ 3, history all below threshold): confidence ~ 0.88 (green)
     - Occasional elevation (NIS ~ 8, some history above): confidence ~ 0.65 (amber)
     - Diverged filter (NIS > CHI2_THRESHOLD): confidence < 0.60 (red)
     This satisfies the visual intent of F-051 without requiring a separate calibration step.
   - Dependencies: Requires step 1 (CHI2_THRESHOLD_6DOF is already defined).
   - Risk: **Low**. The formula is heuristic. Post-POC, a proper credibility score based
     on the chi-squared CDF would be more principled (e.g., `1 - scipy.stats.chi2.cdf(nis, df=6)`).

## Risks and mitigations

- **Risk**: `propagator.py` is not yet implemented (it is a stub as of this plan date).
  `kalman.py` depends on `propagator.tle_to_state_vector_eci_km` in the `predict` function.
  **Mitigation**: All Kalman unit tests mock the propagator. The implementer must implement
  `kalman.py` such that the import of `propagator` is deferred or guarded so that
  `py_compile` and `mypy` pass even if the propagator stub has not been replaced. The import
  at module level is safe since the stub does not raise on import, only on call. The
  integration test (`test_full_update_cycle_with_real_propagator`) must be skipped or
  conditioned on propagator availability.

- **Risk**: FilterPy's `UnscentedKalmanFilter` internal attributes (`ukf.y`, `ukf.S`)
  may differ between filterpy versions or may not be set until after `update()` is called.
  **Mitigation**: The implementer must run `pip show filterpy` to confirm version is 1.4.5+,
  then inspect `filterpy.kalman.UnscentedKalmanFilter` source to verify `y` and `S` are
  set. If they are not available, compute manually: `y = z - ukf.x_prior` and
  `S = ukf.P_prior + ukf.R`.

- **Risk**: Setting `ukf.fx = _fx_sgp4` after construction may not work correctly with
  FilterPy's UKF if it caches sigma-point functions at `__init__` time.
  **Mitigation**: If dynamic assignment fails, subclass `UnscentedKalmanFilter` and
  override the predict method, or construct a new UKF object each predict cycle (acceptable
  for POC given 30-minute update intervals). Flag in implementation comment.

- **Risk**: The sigma-point design (all 13 sigma points map to the same SGP4 output) means
  the UKF covariance prediction is essentially `P_pred = P + Q` rather than `F*P*F^T + Q`.
  This is an approximation.
  **Mitigation**: This is explicitly documented in the design note at step 4. For the POC,
  the Q matrix dominates covariance growth, and the approximation is acceptable. Post-POC,
  replacing SGP4 with a numerical integrator (POST-003) would restore proper sigma-point
  divergence through the dynamics.

- **Risk**: `compute_confidence` formula is heuristic and may not produce values that
  align with the frontend colour thresholds (F-051) across all operating conditions.
  **Mitigation**: The formula is calibrated against the three frontend thresholds in the
  step 9 description. If demo testing reveals the colours are not as expected, the
  constants (2/3 weighting, 2*threshold denominator) can be adjusted without touching
  any other module.

- **Risk**: NF-001 (100ms per update) may be violated if FilterPy's UKF sigma-point
  propagation is slow due to 13 calls into `_fx_sgp4`, each of which calls astropy
  (through `propagator.tle_to_state_vector_eci_km`).
  **Mitigation**: Since all 13 sigma-point evaluations return the same value, `_fx_sgp4`
  can cache the result on the first call within a predict cycle (using `functools.lru_cache`
  keyed on the epoch and TLE strings). The implementer should measure wall-clock time after
  first implementation and add caching only if NF-001 is violated.

## Open questions

1. **Propagator dependency ordering**: This plan depends on `propagator.py` being
   implemented (or mocked). The propagator plan (2026-03-28-propagator.md) is in Draft
   status. Should the implementer start with propagator first and then kalman, or implement
   kalman with full mocking and integrate later? **Recommendation**: Implement propagator
   first. The implementer should not begin Phase 3 of this Kalman plan until propagator
   integration tests pass. Awaiting human decision.

2. **`fx=None` at UKF construction**: FilterPy's `UnscentedKalmanFilter.__init__` may
   require a callable `fx` — passing `None` might raise at construction time. If so, the
   implementer should pass a lambda identity `fx=lambda x, dt: x` at construction and
   override it in `predict`. This is a minor implementation detail but could block Phase 2
   step 3. The implementer must check and handle this without deviation from the plan's
   intent. A comment should be left if the lambda workaround is used.

3. **`get_state` extended keys**: The stub's docstring lists four return keys
   (`state_eci_km`, `covariance_km2`, `last_epoch_utc`, `confidence`). This plan adds
   `nis`, `anomaly_flag`, and `innovation_eci_km` to the return dict (step 7). This is
   not a deviation from the interface contract (the docstring uses "Dict with keys:" not
   "Dict with only keys:"), but a human reviewer should confirm that the API layer
   (`main.py`) consuming `get_state` output is prepared to receive these additional keys.

4. **NIS history cap**: The plan specifies capping `nis_history` at 20 entries. This
   value is not in the requirements. 20 corresponds to approximately 10 hours of history
   at 30-minute update intervals, which is a reasonable window for anomaly classification
   by `anomaly.py`. If `anomaly.py` plans require a longer window, this cap must increase.
   Awaiting `anomaly.py` plan for confirmation.
